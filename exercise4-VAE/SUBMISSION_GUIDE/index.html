    Guia de Submissão - Exercício VAE

    Visão Geral

    Este guia orienta você através de completar e submeter o exercício VAE no GitHub Pages.

    Passo 1: Gerar Resultados

    Execute a implementação principal para gerar visualizações:

    cd docs/exercise4-VAE
python vae.py

    Isto cria um diretório results/ com: - training_loss.png - reconstructions.png - generated_samples.png - latent_space.png - vae_model.pth

    Passo 2: (Opcional) Análise de Crédito Extra

    Comparação VAE vs Autoencoder

    python compare_vae_ae.py

    Gera gráficos de comparação em results/: - comparison_reconstructions.png - comparison_latent_spaces.png - comparison_losses.png

    Efeito da Dimensão Latente

    python latent_dim_analysis.py

    Gera análise de dimensões latentes: - latent_dim_analysis.png - samples_latent_dim_2.png até samples_latent_dim_50.png

    Passo 3: Preparar Seu Relatório

    Crie um arquivo markdown para GitHub Pages com estas seções:

    1. Introdução

    • Breve explicação de VAEs
    • Diferença de Autoencoders padrão
    • Por que aprender sobre VAEs

    2. Arquitetura

    • Estrutura do codificador
    • Estrutura do decodificador
    • Truque de reparametrização
    • Componentes da função de perda

    3. Detalhes de Implementação

    • Conjunto de dados usado (MNIST)
    • Configuração de treinamento
    • Hiperparâmetros
    • Hardware usado

    4. Resultados

    Inclua e explique visualizações:

    Qualidade de Reconstrução

    • Mostre reconstructions.png
    • Analise precisão de reconstrução
    • Discuta artefatos

    Geração de Amostras

    • Mostre generated_samples.png
    • Avalie qualidade de amostras
    • Avalie cobertura de modo

    Organização do Espaço Latente

    • Mostre latent_space.png
    • Interprete estrutura de clusters
    • Discuta organização semântica

    Progresso de Treinamento

    • Mostre training_loss.png
    • Analise convergência
    • Verifique overfitting

    5. Descobertas Principais

    • Desempenho do VAE em MNIST
    • Propriedades do espaço latente
    • Qualidade de amostras geradas
    • Trade-off reconstrução-geração

    6. Desafios e Soluções

    • Desafios encontrados
    • Soluções implementadas
    • Lições aprendidas

    7. Conclusões

    • Resumo de descobertas
    • Efetividade do VAE
    • Melhorias futuras

    8. (Opcional) Crédito Extra

    Se completado, inclua: - Resultados de comparação VAE vs AE - Resultados de análise de dimensão latente - Métricas de desempenho entre configurações

    Passo 4: Organizar Arquivos

    Organize arquivos em seu repositório GitHub Pages:

    repo/
├── index.md (página principal)
├── exercise4-vae.md (ou similar)
├── results/
│   ├── training_loss.png
│   ├── reconstructions.png
│   ├── generated_samples.png
│   ├── latent_space.png
│   └── (arquivos de crédito extra se aplicável)
└── (outros exercícios)

    Passo 5: Referenciar Seu Código

    Documente sua implementação:

    Seções de Código Principais para Explicar

    1. Definição da Classe VAE
    2. Arquitetura Codificador e Decodificador

    3. Passagem direta com reparametrização

    4. Função de Perda

    5. Cálculo de perda de reconstrução
    6. Fórmula de divergência KL

    7. Perda combinada

    8. Loop de Treinamento

    9. Batching de dados
    10. Passo de otimização

    11. Monitoramento de perda

    12. Funções de Visualização

    13. Como reconstruções são geradas
    14. Como amostras são criadas
    15. Como espaço latente é visualizado

    Passo 6: Criar GitHub Pages

    1. Crie repositório GitHub para o curso
    2. Habilite GitHub Pages em Configurações
    3. Crie arquivos markdown com sua análise
    4. Inclua imagens do diretório results/
    5. Use sintaxe ![alt text](path/to/image.png)

    Exemplo markdown:

    ## Qualidade de Reconstrução do VAE

![Reconstruções](results/reconstructions.png)

A qualidade de reconstrução mostra que o VAE aprendeu com sucesso a 
codificar e decodificar dígitos MNIST. As reconstruções são relativamente 
claras com leve desfoque típico de saída VAE.

    Passo 7: Submeter

    1. Certifique-se de que seu site GitHub Pages está ao vivo
    2. Copie a URL do GitHub Pages
    3. Submeta a URL em insper.blackboard.com
    4. Formato: https://username.github.io/repo/exercise4-vae

    Checklist de Avaliação

    Antes de submeter, verifique:

    Implementação (3 pts)

    • Arquitetura VAE implementada corretamente
    • Codificador e Decodificador funcionando
    • Truque de reparametrização implementado
    • Perda computada corretamente (Reconstrução + KL)

    Treinamento & Avaliação (1 pt)

    • Modelo treinado em MNIST
    • Perdas de treinamento e validação monitoradas
    • Pelo menos 20 epochs de treinamento

    Amostragem (2 pts)

    • Amostras geradas do espaço latente
    • Amostras mostram padrões tipo-dígito
    • Múltiplas amostras diversas geradas

    Espaço Latente (2 pts)

    • Visualização do espaço latente criada
    • Classes se agrupam por tipo de dígito
    • Transições suaves entre clusters

    Visualizações (1 pt)

    • Todos os gráficos necessários presentes
    • Rótulos e títulos claros
    • Aparência profissional

    Relatório (1 pt)

    • Explicação escrita clara
    • Estrutura organizada
    • Interpretação de resultados

    Crédito Extra Opcional

    • Comparação VAE vs AE (se realizado)
    • Análise de dimensão latente (se realizado)
    • Experimentos adicionais (se realizado)

    Problemas Comuns na Submissão

    Problema: Imagens não carregam no GitHub Pages

    • Certifique-se de que caminhos são relativos
    • Verifique que imagens estão no repositório
    • Verifique que .gitignore não exclui imagens

    Problema: Problemas de formatação Markdown

    • Valide sintaxe markdown
    • Verifique sintaxe de imagem: ![alt](path/image.png)
    • Use hierarquia apropriada de cabeçalhos

    Problema: Referências de código pouco claras

    • Inclua snippets de código no relatório
    • Explique detalhes principais de implementação
    • Vincule ao repositório GitHub se necessário
