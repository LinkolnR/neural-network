    Isto treina um VAE em MNIST e gera 4 arquivos de visualização em ./results/

    O que foi Implementado

    Componentes Principais

    1. Rede Codificadora
    2. Entrada: 784 (imagens 28x28 achatadas)
    3. Oculta: 512 → 256 neurônios

    4. Saída: mu e logvar (parâmetros da distribuição latente)

    5. Rede Decodificadora

    6. Entrada: latent_dim (padrão 20)
    7. Oculta: 256 → 512 neurônios

    8. Saída: 784 (imagem reconstruída)

    9. Truque de Reparametrização

    10. Permite fluxo de gradiente através de amostragem estocástica

    11. Fórmula: z = mu + std * epsilon

    12. Função de Perda

    13. Perda de Reconstrução (BCE)
    14. Divergência KL (regularização)
    15. Total: Reconstrução + KL

    Funcionalidades

    • Carregamento de dados com divisão treino/validação/teste
    • Loop de treinamento com monitoramento
    • Visualização de reconstrução
    • Geração de amostras do espaço latente
    • Projeção 2D do espaço latente (t-SNE)
    • Curvas de perda de treinamento

    Hiperparâmetros Principais

    Podem ser modificados no código:

    • latent_dim: 20 (padrão)
    • epochs: 20 (padrão)
    • batch_size: 64 (padrão)
    • learning_rate: 1e-3 (padrão)
    • dataset: 'MNIST' ou 'FashionMNIST'

    Detalhes da Arquitetura

    Decomposição da Perda do VAE

    Perda Total = Perda de Reconstrução + Divergência KL

Perda de Reconstrução = Entropia Cruzada Binária(x, x_recon)
                      = Σ[x*log(x_recon) + (1-x)*log(1-x_recon)]

Divergência KL = -0.5 * Σ[1 + logvar - mu² - exp(logvar)]
               = Mede diferença de N(0,1)

    Dinâmica de Treinamento

    • Época 0-5: Diminuição rápida de perda, reconstrução melhora
    • Época 5-15: Ajuste fino, espaço latente se organiza
    • Época 15-20: Convergência, perda de validação platô

    Capacidade do Modelo

    • Parâmetros totais: ~400K
    • Codificador: ~300K
    • Decodificador: ~300K
    • Compartilhado: Mínimo

    Expectativas de Desempenho

    • Perda de Reconstrução: ~60-80 (por amostra)
    • Divergência KL: ~2-5 (por amostra)
    • Tempo de Treinamento: ~2-5 min (CPU), ~30 seg (GPU)

    Entendendo os Resultados

    Boa Reconstrução

    • Imagens original e reconstruída são similares
    • Dígitos claramente visíveis em reconstruções
    • Sem distorções maiores

    Boa Geração

    • Amostras geradas mostram padrões de dígitos
    • Cobrem múltiplos tipos de dígitos
    • Clareza razoável

    Bom Espaço Latente

    • Dígitos da mesma classe se agrupam
    • Transições suaves entre clusters
    • Separação clara de classes

    Problemas Comuns e Soluções

    Problema Solução
    Reconstruções desfocadas Treinar mais longo, aumentar capacidade
    Espaço latente colapsado Aumentar peso de KL, usar beta-VAE
    Geração ruim Mais dimensões latentes, treinamento mais longo
    Colapso de modo Melhor regularização, variar dim latente
